## Why this order
Foundation first (everything needs realtime + auth + the LLM interface stub). Master/reference data (5e Reference) before the things that point at it. Characters before Inventory/Combat (both reference a character). Combat тнР once Characters + Monsters exist. Story after Combat (sessions log encounters). Player UI/Dice/Dashboard reads from everything, so it's late. AI last тАФ it generates **into** the 5e/Character/Story entities, which must exist first.

> 5e makes schema heaviest at **Characters + Spells** (slots by level, conditions, proficiency, saves) тАФ expect Sprints 1тАУ2 to carry more schema than a typical app.
