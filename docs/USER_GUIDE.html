D&D Campaign Manager

เครื่องมือจัดการแคมเปญ D&D 5e แบบ real-time สำหรับทั้งโต๊ะ — ไม่ต้องสมัครสมาชิก ไม่ต้องอินเทอร์เน็ต ใช้ได้บน LAN เดียวกัน

🖥️ Self-hosted บนเครื่อง DM 🌐 ผู้เล่นเข้าผ่าน Browser ⚡ Real-time Socket.io 🆓 ฟรี ไม่มีค่าใช้จ่าย
ภาพรวม

ระบบทำอะไรได้บ้าง?

ครอบคลุม gameplay loop ของ D&D 5e ตั้งแต่สร้างแคมเปญจนถึงจบ session

👑 Dungeon Master (DM)

สร้างแคมเปญ · จัดการ encounter · ควบคุม combat tracker · บันทึก session/quest/NPC · ใช้ AI ช่วยเขียน

เข้าร่วมด้วย invite code · สร้างและแก้ตัวละคร 5e · โยนลูกเต๋าแบบ broadcast · ดู HP/slots/conditions ตัวเอง

เริ่มต้น

เริ่มใช้งาน

เปิดเซิร์ฟเวอร์ · สร้างแคมเปญ · ผู้เล่นเข้าร่วม ใช้เวลาไม่ถึง 2 นาที

DM — เปิดเซิร์ฟเวอร์

1
รันเซิร์ฟเวอร์
เปิด terminal ที่โฟลเดอร์โปรแกรม แล้วรัน npm run dev (dev mode) หรือ npm start (production) เซิร์ฟเวอร์จะขึ้นที่ port 3000
2
เปิด browser ไปที่ http://localhost:3000
เห็นหน้าหลักมี 2 ปุ่ม: Create Campaign (สำหรับ DM) และ Join Campaign (สำหรับ Player)
3
บอก IP ให้ผู้เล่น
ผู้เล่นเข้าที่ http://<ip-ของเครื่อง-DM>:3000 เช่น http://192.168.1.5:3000 ต้องอยู่บน Wi-Fi เดียวกัน

DM — สร้างแคมเปญ

1
กด "Create Campaign"
จะเปิดฟอร์มให้กรอก Campaign Name และชื่อ DM
2
กรอกชื่อแคมเปญ + ชื่อ DM → กด Create
ระบบสร้างแคมเปญและพา DM เข้าหน้า campaign dashboard อัตโนมัติ
3
แชร์ Invite Code ให้ผู้เล่น
รหัสเชิญมีรูปแบบเช่น K7Q-M2P แสดงบนหน้า campaign ก็อปไปบอกผู้เล่นได้เลย
✅

Session token ของ DM เก็บใน localStorage — ถ้า browser ปิดแล้วเปิดใหม่ DM จะยังได้ session เดิมคืนมา (ไม่หายแน่นอน)

Player — เข้าร่วมแคมเปญ

1
เปิด browser ไปที่ IP ของ DM
เช่น http://192.168.1.5:3000 → กด "Join Campaign"
2
ใส่ Invite Code + ชื่อที่แสดง
ชื่อต้องไม่ซ้ำกับคนอื่นในห้อง ระบบจะแจ้งเตือนถ้าชื่อซ้ำ
3
กด Join → เข้าสู่ห้อง
DM และ player คนอื่นจะเห็นชื่อขึ้นใน roster ทันที (real-time ≤ 500ms)
⚠️

ถ้าเชื่อมต่อขาด browser จะ reconnect อัตโนมัติและโหลด state ล่าสุดจากเซิร์ฟเวอร์คืนมา ไม่ต้องทำอะไรเพิ่ม

ตัวละคร (Character Sheet)

สร้างตัวละคร D&D 5e เต็มรูปแบบ — ระบบ auto-fill ค่าตามกฎให้อัตโนมัติ

ผู้เล่นแต่ละคนสร้างตัวละคร 1 ตัวต่อแคมเปญ ระบบรองรับทุกคลาสและ subclass จาก SRD 5.1 + community OGL

สร้างตัวละครใหม่

1
กด "New Character" ในหน้า campaign
เปิด wizard หลายขั้น — กรอกทีละ step ได้เลย
2
เลือก Race
มี 13 race + subraces จาก SRD; ระบบเติม speed, traits, stat modifier อัตโนมัติ
3
เลือก Class + Level
12 คลาส × progression 1–20; ระบบคำนวณ HP, proficiency bonus, spell slots, features ราย level ให้ทันที
4
เลือก Subclass (เมื่อถึง level)
109 subclass (12 SRD + 97 community OGL) เลือกแล้วระบบเติม feature ของ subclass ตาม level อัตโนมัติ
5
เลือก Background (optional)
42 background — เพิ่ม skill/tool proficiency และ feature พิเศษ
6
กรอก Ability Scores + Skill Choices
ใส่ค่า 6 ability (STR/DEX/CON/INT/WIS/CHA) และเลือก skill proficiency ตามจำนวนที่คลาส/background ให้; modifier/saves คำนวณอัตโนมัติ

แท็บบนชีตตัวละคร

⚔️ Core

Ability scores · Saving throws · Skill checks · Passive Perception · Proficiency Bonus · Initiative

❤️ Combat

HP (current/max) · AC · Speed · Hit Dice · Death saves · Attacks

✨ Features

Features ทุกอย่างตาม level จาก Class + Subclass + Race + Background

🔮 Spells

รายการ spell ที่รู้/เตรียมไว้ · Spell slots ราย level · Spellcasting ability

Player

คลังไอเทม (Inventory)

จัดการไอเทม · ใส่/ถอดอุปกรณ์ · attunement · เงิน

เพิ่มไอเทมใหม่

1
ไปที่แท็บ Inventory ในชีตตัวละคร
มีส่วน "Add Item" ให้ค้นหาจากคลัง SRD หรือเพิ่มไอเทมเองแบบ custom
2
ค้นหาหรือกรอกชื่อ → เลือก item
ระบบดึงข้อมูลจาก SRD: weight, properties, damage, rarity ให้อัตโนมัติ; แก้ได้ถ้าต้องการ
3
ระบุ quantity และ notes → กด Add
ไอเทมปรากฏในรายการ; จากนั้น equip/unequip หรือตั้งค่า attunement ได้

Attunement (กติกา 5e)

ไอเทมที่ต้อง attune จะมีเครื่องหมาย ◈ — attuned ได้สูงสุด 3 ชิ้นพร้อมกัน ถ้าพยายาม attune ชิ้นที่ 4 ระบบจะแจ้ง error attunement_limit

การต่อสู้ (Combat Tracker)

Initiative · HP · conditions · turn order — sync real-time ทุกหน้าจอ

Combat tracker คือหัวใจของระบบ — DM คุมทุกอย่าง ผู้เล่นดู HP/conditions/initiative ของตัวเองและ NPC ได้แบบ live

เปิด Encounter ใหม่

1
DM กด "New Encounter"
ตั้งชื่อ encounter (เช่น "Goblin Ambush at the Bridge") แล้วกด Create
2
เพิ่มผู้ร่วมต่อสู้ (Combatants)
Player Characters: เพิ่มจาก character list ได้เลย (HP ดึงมาจากชีตอัตโนมัติ)
Monsters/NPCs: ค้นหาจากคลัง SRD หรือสร้าง custom combatant
3
กรอก Initiative
DM กรอก initiative ของแต่ละ combatant (จะใช้ลูกเต๋าตัวจริงที่โต๊ะ หรือใช้ dice roller ในโปรแกรมก็ได้) ระบบจัดเรียงให้อัตโนมัติ
4
กด "Start Encounter"
ระบบ broadcast turn order ไปทุกหน้าจอ ผู้เล่นเห็น initiative track ทันที

เรื่องราว (Story Module)

Session logs · Quests · NPCs · Journal — เก็บประวัติแคมเปญทั้งหมดในที่เดียว

📅 Session Logs

บันทึกแต่ละครั้งที่เล่น — วันที่, summary/recap ที่ DM เขียน, encounters ที่เกิดขึ้น

⚔️ Quests

สร้างและติดตาม quest — ชื่อ, ผู้มอบ, objectives, reward; สถานะ: active / completed / failed

🧝 NPCs

บันทึก NPC สำคัญ — ชื่อ, appearance, personality, secret, faction, CR

📓 Journal

บันทึกอิสระ — event, clue, lore ที่ DM จดไว้; ค้นหาได้ผ่าน global search

สร้าง Session Log

1
กด "New Session" ในแท็บ Story
ระบุวันที่เล่น (default = วันนี้) และชื่อ session (optional)
2
เขียน Recap / Summary
เขียนได้เองหรือใช้ AI DM Assistant ช่วยสร้าง recap draft ก็ได้ (ดูหัวข้อ AI)
3
กด Save — session ปรากฏในรายการ
session เก่าๆ ยังดูย้อนหลังได้เสมอ

สถานะ Quest

active — กำลังดำเนินการ completed — สำเร็จแล้ว failed — ล้มเหลว

🎲 Dice Roller

DM เท่านั้น · AI Features

✦ AI DM Assistant

สร้าง NPC · loot · quest · recap ด้วย AI — DM อนุมัติทุก draft ก่อนบันทึกเข้าแคมเปญ

✦

AI ช่วยเขียน prose เท่านั้น — XP, loot tier, spell slots, dice คำนวณโดย code ฝั่ง server ไม่ใช้ AI คำนวณ ผลลัพธ์จึงถูกต้องตามกฎ 5e เสมอ ไม่ขึ้นกับ model

Providers ที่รองรับ

🟢
Ollama (แนะนำ) — รันบนเครื่อง DM เอง ฟรี ไม่ส่งข้อมูลออก
ติดตั้ง Ollama → ดาวน์โหลด model (ค่าเริ่มต้น qwen2.5:14b) → รัน Ollama ก่อนเปิดเกม
📋
Import from Claude (ใช้ได้เสมอ) — paste ข้อความจาก Claude/ChatGPT ใดๆ มาแปลงเป็น draft
ไม่ต้องรัน Ollama · ทำงานแบบ offline ได้
⚠️

ถ้า Ollama ไม่ได้รัน ระบบจะแสดง banner "Ollama not connected" ในแพนเนล AI — แต่ Import ยังใช้ได้เสมอ และ feature อื่นทุกอย่างในโปรแกรมทำงานปกติ ไม่ต้องมี AI

ประเภท Draft ที่สร้างได้

🧝 NPC Draft

ระบุ prompt เช่น "tiefling merchant สงสัยน่าไว้ใจ CR 3" — ได้ชื่อ, หน้าตา, นิสัย, ความลับ; Approve → NPC บันทึกใน Story

💎 Loot Draft

ระบุ CR และ prompt — XP คำนวณจาก crToXp() (code ไม่ใช่ AI), tier กำหนดจาก CR bracket; AI เขียนแค่ชื่อไอเทมและ flavor text

📜 Quest Draft

ระบุ theme เช่น "party เพิ่งเคลียร์ dungeon ให้ hook ต่อ" — ได้ชื่อ quest, objectives, reward hint; Approve → Quest บันทึกใน Story

📰 Session Recap

ระบุ encounters ที่เกิดขึ้นใน session — AI เขียน opening narration / recap; Approve → บันทึกใน Session.summary

Flow: Generate → Review → Approve

1
เลือกประเภท → กรอก prompt → กด Generate
Prompt สูงสุด 2,000 ตัวอักษร สำหรับ loot ต้องระบุ CR ด้วย

เซฟ · เล่นต่อ · Backup

ข้อมูลทุกอย่างเซฟอัตโนมัติลงฐานข้อมูล — เปิดมาวันหลังก็เล่นต่อได้ ไม่ต้องกด Save

1. เซฟอัตโนมัติ — ไม่มีปุ่ม Save

ทุกการกระทำเขียนลง SQLite (prisma/dev.db) ทันที ไม่ต้องกดบันทึก สิ่งที่อยู่ถาวร:

✅ อยู่ถาวรในฐานข้อมูล

ตัวละคร (รวมที่สร้างใหม่), HP/สถานะ/condition, item & spell, currency, Story (session/quest/NPC/journal), ผล combat & dice

⚡ อยู่แค่ตอน server รัน (live state)

ใครออนไลน์อยู่ (presence), socket connection — สร้างใหม่อัตโนมัติเมื่อ reconnect ไม่ใช่ข้อมูลที่ต้องเซฟ

2. เล่นต่อวันถัดไป

1
เปิด Ollama (ถ้าจะใช้ AI) แล้วรัน npm run dev
server โหลดฐานข้อมูล + rehydrate แคมเปญที่ยัง active กลับมาให้เองอัตโนมัติ
2
เปิด URL แคมเปญเดิม
browser จำ session token ไว้ใน localStorage (dnd.sessions) → เชื่อมต่อกลับให้เอง ไม่ต้องใส่ invite code ซ้ำ และเข้า role เดิม (DM / player)
3
ตัวละครที่สร้างเมื่อวานยังอยู่ครบ
เพราะอยู่ในฐานข้อมูล ไม่ใช่แค่ใน RAM — เล่นต่อได้ทันที

3. Rejoin-by-name — กู้ตัวละครคืนเมื่อ session หาย

ถ้า localStorage หาย / เปลี่ยน browser / ล้าง cache แค่กด Join ด้วย ชื่อผู้เล่นเดิม ในแคมเปญเดิม → ระบบเชื่อมคุณกลับเข้า session เดิม พร้อมตัวละครที่ claim ไว้ (ไม่สร้าง seat ซ้ำ, ไม่ขึ้น error ชื่อซ้ำ)

ชื่อผู้เล่นเดิม → ได้ตัวละครเดิมคืน case-insensitive (Thorin = thorin) ชื่อ DM ถูกสงวน — สวมสิทธิ์ DM ไม่ได้
🔐

Trust model แบบ LAN: invite code คือความลับที่แชร์กัน, ส่วนชื่อ = ตัวตน — เหมาะกับโต๊ะที่เล่นกับคนที่ไว้ใจ จึงไม่ต้องมี password

4. Export / Import Backup

สำหรับ สำรองข้อมูล (เผื่อ DB พัง) หรือ ย้ายไปเครื่องอื่น / แชร์ให้คนอื่น — ไม่จำเป็นสำหรับเล่นต่อในเครื่องเดิม

📤 Export (DM เท่านั้น)

ในฝั่ง DM ที่กล่อง invite code → ปุ่ม "Export backup" ดาวน์โหลดเป็นไฟล์ campaign-<ชื่อ>-<วันที่>.json

📥 Import / Restore

หน้าแรก → "Restore from backup" (หรือเปิด /import) → เลือกไฟล์ → สร้างแคมเปญใหม่พร้อม invite code ใหม่
